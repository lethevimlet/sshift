Folder Drag & Drop - Manual Test Instructions

Prerequisites:
  • Server running at http://localhost:3000
  • At least 2 bookmarks created
  • At least 1 folder created

Test 1: Drag Bookmark to Folder

Steps:

  1. Open http://localhost:3000 in browser
  2. Locate a bookmark in the root (not in a folder)
  3. Drag the bookmark and drop it on a folder header

Expected Result:

Bookmark moves into the folder and appears indented under the folder name.

Console should show: "Moving bookmark [id] to folder [folderId]"

Test 2: Drag Bookmark from Folder to Root

Steps:

  1. Expand a folder that contains bookmarks
  2. Drag a bookmark from the folder
  3. Drop it on the empty space below all folders (root area)

Expected Result:

Bookmark moves out of the folder and appears in the root area.

Console should show: "Dropping to root: [bookmark name]"

Test 3: Reorder Bookmarks in Same Folder

Steps:

  1. Create at least 2 bookmarks in the same folder (or root)
  2. Drag one bookmark and drop it on another bookmark in the same location

Expected Result:

Bookmarks reorder within the same folder. Order is persisted after page refresh.

Console should show: "Same folder reordering. Dragged index: X, Target index: Y"

Test 4: Drag Bookmark Between Folders

Steps:

  1. Create at least 2 folders with bookmarks in each
  2. Drag a bookmark from one folder
  3. Drop it on another folder's header

Expected Result:

Bookmark moves from first folder to second folder.

Console should show: "Moving to different folder: [folderId]"

Test 5: Drag Folder to Reorder

Steps:

  1. Create at least 2 folders
  2. Drag one folder header and drop it on another folder header

Expected Result:

Folders reorder. Order is persisted after page refresh.

Test 6: Visual Feedback During Drag

Steps:

  1. Start dragging a bookmark
  2. Hover over different drop targets without dropping

Expected Result:

Visual feedback should appear:

  • When hovering over a bookmark: Blue border on top
  • When hovering over a folder header: Highlighted background with blue border
  • When hovering over root area: Light background highlight

Test 7: Persistence After Page Reload

Steps:

  1. Perform any of the above drag operations
  2. Refresh the page (F5)

Expected Result:

All changes should persist - bookmarks remain in their new folders/positions.

Test 8: Empty Root Drop Zone

Steps:

  1. Move all bookmarks into folders (so root is empty)
  2. Drag a bookmark from a folder
  3. Drop it on the empty root area

Expected Result:

Bookmark successfully moves to root. Root container should always be available as a drop target even when empty.

Debug Console Commands

Open browser console (F12) and run these commands to check state:

// Get all bookmarks
fetch('/api/bookmarks').then(r => r.json()).then(console.log)

// Get all folders
fetch('/api/folders').then(r => r.json()).then(console.log)

// Check config.json directly (in server terminal)
cat config.json

Known Issues to Check

  • ✓ Root container should always be droppable (even when empty)
  • ✓ Dropping on bookmark in root should not trigger "move to root"
  • ✓ Visual feedback should clear after drag ends
  • ✓ Folder expansion state should persist
