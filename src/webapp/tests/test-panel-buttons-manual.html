Manual Test: Font Size & Special Keys Buttons in Multi-Panel Layouts

Test Objective

Verify that the font size increase/decrease buttons and special keys button work correctly in all panel layouts, not just single panel layout.

Prerequisites

  • SSHIFT server running on localhost:3000
  • At least one SSH connection configured (for special keys test)

Test 1: Font Size Buttons in Single Panel

Steps:
  1. Open the application in single panel layout (default)
  2. Note the current font size (check console: app.terminalFontSize)
  3. Click the - button (decrease font size)
  4. Observe the terminal font size decreases
  5. Click the + button (increase font size)
  6. Observe the terminal font size increases
Expected Result: Font size buttons work correctly in single panel layout. All terminals update their font size.

Test 2: Font Size Buttons in Multi-Panel Layout

Steps:
  1. Switch to 2-column layout (click layout button in header)
  2. Note the current font size
  3. Click the - button in panel-0 (left panel)
  4. Observe ALL terminals decrease font size (not just panel-0)
  5. Click the + button in panel-1 (right panel)
  6. Observe ALL terminals increase font size
  7. Switch to 2-rows layout
  8. Repeat font size button tests in both panels
  9. Switch to 2x2 grid layout
  10. Repeat font size button tests in all four panels
Expected Result: Font size buttons work in ALL panels across ALL layouts. Font size is global and affects all terminals.

Test 3: Special Keys Button in Single Panel

Steps:
  1. Return to single panel layout
  2. Open an SSH connection
  3. Click the keyboard button (special keys)
  4. Observe the special keys modal opens
  5. Click a special key (e.g., Ctrl+C)
  6. Observe the key is sent to the active SSH session
Expected Result: Special keys modal opens and keys are sent to the active SSH session.

Test 4: Special Keys Button in Multi-Panel Layout

Steps:
  1. Switch to 2-column layout
  2. Open an SSH connection in panel-0 (left panel)
  3. Open another SSH connection in panel-1 (right panel)
  4. Click the keyboard button in panel-0
  5. Observe the special keys modal opens
  6. Click Ctrl+C and verify it goes to panel-0's session
  7. Close the modal
  8. Click the keyboard button in panel-1
  9. Observe the special keys modal opens
  10. Click Ctrl+D and verify it goes to panel-1's session
Expected Result: Special keys button works in ALL panels. Each panel's special keys button sends keys to that panel's active session.
Note: If a panel has no active session, clicking the special keys button should show a warning toast: "No active session in this panel"

Test 5: Edge Cases

Steps:
  1. In multi-panel layout, click special keys button on an empty panel (no session)
  2. Observe warning toast appears
  3. Open an SFTP session in a panel
  4. Click special keys button on that panel
  5. Observe warning toast: "Special keys only work in SSH sessions"
  6. Switch between layouts rapidly while clicking font size buttons
Expected Result: Appropriate error messages shown. No JavaScript errors in console. Buttons remain functional across layout changes.

Console Commands

Use these commands in the browser console to verify functionality:

  • app.terminalFontSize - Check current font size
  • app.activeSessionsByPanel - See active sessions per panel
  • app.activeSessionId - Check current active session
  • app.setLayout('2-columns') - Switch to 2-column layout
  • app.setLayout('single') - Switch to single panel layout

Implementation Details

Changes Made:

  • Created handleFontSizeChange(panelId, delta) method to handle font size changes from any panel
  • Created handleSpecialKeys(panelId) method to handle special keys button from any panel
  • Font size is global - changes affect all terminals across all panels
  • Special keys are panel-specific - each panel's button targets that panel's active session
  • Proper error handling for empty panels and non-SSH sessions
