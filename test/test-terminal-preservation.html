Terminal Preservation Test

This test verifies that terminal content is preserved when switching layouts.

Test Instructions:

  1. Open the SSHIFT application in a browser
  2. Connect to an SSH server and run some commands (e.g., ls -la, pwd)
  3. Switch to a multi-panel layout (e.g., "2 Columns")
  4. Verify that the terminal content is still visible
  5. Switch back to single layout
  6. Verify that the terminal content is still visible

Expected Behavior:

  • Terminal content should be preserved when switching layouts
  • Terminal should be properly resized to fit the new panel
  • Active tab should be restored after layout change
  • Non-active terminals should be hidden but their content preserved

Console Logs to Check:

[SSHIFT] Stored X tabs before layout change [SSHIFT] Preserved terminal wrapper for session: ssh-XXXXX [SSHIFT] Reused preserved terminal wrapper for session: ssh-XXXXX [SSHIFT] Fitting preserved terminal for session: ssh-XXXXX [SSHIFT] Preserved terminal fitted successfully for session: ssh-XXXXX [SSHIFT] Restoring active tab: ssh-XXXXX

Common Issues:

  • Black terminal: Terminal wrapper was not preserved, or terminal.open() was called on an already-open terminal
  • Terminal not visible: Active class not properly set on the wrapper
  • Terminal too small/large: Fit addon not called after layout change
