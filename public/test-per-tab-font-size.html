Per-Tab Font Size Independence Test

Overview

This test verifies that font size changes are now independent per tab, rather than global across all terminals.

Key Change: Each session (tab) now maintains its own font size. The -/+ buttons only affect the active tab in that panel.

Test 1: Single Panel - Multiple Tabs

Steps:
  1. Open the application in single panel mode
  2. Connect to a server (Tab 1)
  3. Open a second connection (Tab 2)
  4. Switch to Tab 1 and increase font size using + button (3-4 times)
  5. Switch to Tab 2 - font size should be default
  6. Increase font size in Tab 2 (2 times)
  7. Switch back to Tab 1 - font size should still be larger
Expected Result:
  • Each tab maintains its own font size
  • Switching between tabs preserves their individual font sizes
  • Font size changes only affect the active tab

Status: PENDING

Test 2: Multi-Panel Layout

Steps:
  1. Switch to 2-panel layout (horizontal or vertical split)
  2. Connect to a server in Panel 1 (Tab 1)
  3. Connect to a server in Panel 2 (Tab 2)
  4. In Panel 1, increase font size using + button
  5. In Panel 2, font size should remain default
  6. Decrease font size in Panel 2
  7. Panel 1's font size should be unchanged
Expected Result:
  • Each panel's active tab has independent font size
  • Font size changes in one panel don't affect other panels
  • -/+ buttons work correctly in all panel layouts

Status: PENDING

Test 3: Font Size Persistence During Session

Steps:
  1. Open multiple tabs with different font sizes
  2. Set Tab 1 to font size 18
  3. Set Tab 2 to font size 14
  4. Set Tab 3 to font size 20
  5. Cycle through all tabs multiple times
  6. Verify each tab retains its font size
Expected Result:
  • Tab 1 consistently shows font size 18
  • Tab 2 consistently shows font size 14
  • Tab 3 consistently shows font size 20
  • No "bleed" of font size between tabs

Status: PENDING

Test 4: New Tab Font Size

Steps:
  1. Open Tab 1 and increase font size to 20
  2. Open a new Tab 2
  3. Tab 2 should start with default font size
  4. Tab 1 should still have font size 20
Expected Result:
  • New tabs start with the default font size
  • Existing tabs maintain their font sizes
  • Font size is not inherited from other tabs

Test 5: Special Keys Dialog

Steps:
  1. Open application in multi-panel layout
  2. Click "Special Keys" button in Panel 1
  3. Dialog should appear and work correctly
  4. Click "Special Keys" button in Panel 2
  5. Dialog should appear and work correctly
  6. Verify buttons work in all panel layouts
Expected Result:

  • Special keys work correctly in all panels

Test 6: SFTP Sessions

Steps:
  1. Open an SFTP session
  2. Increase font size using + button
  3. Open another SFTP or SSH session
  4. Verify font sizes are independent
Expected Result:
  • SFTP sessions support per-tab font size
  • Font size changes only affect the active SFTP tab
  • Switching between SFTP tabs preserves font sizes

Status: PENDING

Implementation Details

  • session.fontSize - Each session now stores its own font size
  • setSessionFontSize(sessionId, size) - New method to set font size for specific session
  • handleFontSizeChange(panelId, delta) - Updated to only affect active tab in panel
  • switchTab() - Restores font size when switching tabs
  • Font size is initialized from this.terminalFontSize (default) when creating new sessions

Console Verification

Open browser console and check for these log messages:

[SSHIFT] Font size set to 18 for session session-123
[SSHIFT] Restored font size 18 for session session-123
[SSHIFT] Font size change for panel: panel-left delta: 1

These messages confirm per-session font size is working correctly.
